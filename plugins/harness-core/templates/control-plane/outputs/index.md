# Outputs

Durable artifacts produced by workstreams. Organized by workstream name. This folder survives workstream cleanup — the workstream folder is deleted at cleanup; this folder is not.

**Rule:** Any file a workstream produces for use beyond the workstream itself (slides, documents, configs, reports, generated content) goes here, not in the workstream folder. Working files (`_working/`), process artifacts (`plan.md`, `research.md`, `design.md`), and the workstream log entry are the only things that belong inside the workstream folder structure.
